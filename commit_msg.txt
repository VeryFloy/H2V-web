perf: instant startup via stale-while-revalidate cache (like Telegram)

Eliminates the 2-3s blank screen on page reload:

- appCache.ts: localStorage cache utility with LRU eviction for messages
  (max 20 chats × 50 messages, prefix h2v_c_*)
- auth.store: user read from cache synchronously at module init — spinner
  is skipped entirely on reload if user data is already cached
- chat.store: loadChats() renders cached chat list immediately, then
  refreshes from API in background without blocking the UI
- chat.store: loadMessages() pre-fills the message list from cache before
  the API responds — chat opens instantly on reload
- Cache kept fresh by addMessage, addChat, removeChat real-time handlers
- Cache cleared on logout via appCache.clearAll()

Flow after this change:
  Before: JS load → spinner (API) → empty list (API) → messages (API)
  After:  JS load → instant render from cache → silent background refresh
